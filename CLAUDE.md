# 开发准则（本项目）

## 产品定位

StablePulse：稳定币生态动态追踪平台。**动态数据追踪是核心价值，静态内容是辅助。**

## 测试优先级

从高到低：

1. **数据源可达性** — 外部 API（CoinGecko、Basescan、FRED 等）、链上数据是否能拿到
2. **数据完整性与新鲜度** — 字段正确、时效性符合预期
3. **API 路由结构** — 接口返回格式符合预期 schema
4. **组件渲染** — 页面能跑通，关键内容存在（最低优先，不写也可接受）

## 数据追踪规范

- 新增数据源必须先有可达性测试，再上线
- API 路由的响应 schema 必须有对应测试
- x402 payTo 地址抓取等链上逻辑，测试覆盖是上线前提
